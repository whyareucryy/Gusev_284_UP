-----------------Название программы-----------------

ComfortFurniture — информационная система для управления продукцией мебельной компании.

-----------------Стек технологий--------------------

-----------------------Backend----------------------

- Язык программирования: C# (.NET 8.0);
- фреймворк: ASP.NET Core MVC;
- ORM: Entity Framework Core;
- СУБД: Microsoft SQL Server;
- сервисы: Dependency Injection, собственный сервис расчёта сырья.

----------------------Frontend----------------------

- Стилизация: Bootstrap 5, кастомные CSS-классы;
- шрифты: Candara (основной), Google Fonts как fallback;
- цветовая схема: в соответствии с руководством по стилю (#FFFFFF, #D2DFFF, #355CBD);
- клиентская логика: JavaScript (ES6), Fetch API для AJAX-запросов.

---------------Инструменты разработки---------------

- Среда разработки: Visual Studio 2022;
- система контроля версий: Git (репозиторий GitHub/GitLab);
- управление пакетами: NuGet.

--------Функциональные возможности приложения--------

1. Модуль продукции (основной интерфейс).
- Просмотр списка всей продукции в виде таблицы с колонками: артикул, тип продукции, наименование, минимальная стоимость для партнёра, общее время изготовления (суммарно по цехам);
- добавление нового продукта через модальное окно с валидацией полей (артикул, тип, наименование, стоимость, основной материал);
- редактирование существующего продукта — модальное окно автоматически заполняется данными выбранной записи, после сохранения список обновляется;
- переход к просмотру цехов для конкретного продукта.

2. Модуль цехов.
Отображение списка цехов, задействованных в производстве выбранного продукта, с указанием:
- названия цеха;
- количества человек для производства;
- времени изготовления в данном цехе.
- навигация обратно к списку продукции.

3. Модуль расчёта сырья.
Форма для ввода параметров:
- тип продукции (выпадающий список, подгружается из БД);
- тип материала (выпадающий список из БД);
- количество продукции (целое);
- два вещественных параметра (например, габариты).

Расчёт количества необходимого сырья с учётом коэффициента типа продукции и процента потерь материала.
Отображение результата (целое число) либо сообщения об ошибке (-1) при некорректных данных.
Использование отдельного сервиса RawMaterialCalculator, зарегистрированного в DI.
